# Agent 自驱动游戏模式

> 相关索引：[[README]]  
> 相关文档：[[tech-stack]]、[[game-design]]

## 是什么

一种**开发工作流**，与游戏运行时无关。目的是让 AI 编程助手在开发过程中替代人工手动操作，通过命令行逐步推进游戏——验证逻辑、探索地图行为、复现特定场景——而无需打开浏览器或手动构造 HTTP 请求。

---

## 为什么这样设计

**原子操作，无长驻进程。** 每条命令独立运行、完成即退出。AI 助手只需按序发命令，不需要维护任何连接或会话，失败了重试同一条命令即可，没有"卡住"的可能。

**不可变快照，而非可变状态。** 每次操作都把新状态写入带时间戳的独立文件，旧文件永不修改。历史操作序列完整保留，可以随时回退到任意一步重新分叉，也可以对比前后状态变化。

**直接复用游戏逻辑层。** 业务逻辑与传输层（HTTP）彻底分离，CLI 工具和 Web 服务器走同一套代码路径。这意味着在 CLI 里验证通过的逻辑，在 Web 端同样成立，不存在"测试环境与生产环境行为不一致"的问题。

---

## AI 助手如何使用

工作流程极简：读取当前状态 → 选择动作 → 执行并观察结果 → 循环。每次操作的输出包含完整的地图快照、玩家状态和游戏日志，足够 AI 助手在无额外上下文的情况下规划下一步。

这与开发者手动测试本质相同，只是由 AI 助手代为执行命令。
