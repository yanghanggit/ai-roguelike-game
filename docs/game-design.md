# 游戏设计

## 核心机制

游戏是**扫雷式揭格**玩法，而非传统 Roguelike 的移动探索。玩家没有位置概念，所有交互均通过点击格子完成。

- 所有格子初始隐藏，点击后揭开，显示其内容。
- 揭开一格 = 消耗一回合，已揭开的格子再次点击无效。

## 地图

- 地图固定为 **3×3** 或 **4×4** 两种规格。
- 格子类型：`floor`（空地）、`wall`（墙）、`entrance`（入口）、`monster`（怪物）、`treasure`（宝箱）、`item`（道具）、`special`（特殊）。
- 3×3 地图含 ≥1 个入口，4×4 含 ≥2 个入口。

## 关卡结构

- 多张地图组成**树形结构**，玩家从根节点出发。
- 叶节点地图含有最终 Boss。
- `entrance` 格子是通往子地图的连接点。

## 玩家

- 有 HP、最大HP、攻击、防御、等级、经验值属性。
- 当前版本：点击任意格仅记录日志，战斗/道具等具体效果待后续实现。

## 怪物行为节奏

怪物具有 AI 驱动的对话上下文，其行为按以下节奏展开：

- **游戏开始**：所有怪物静默完成一次初始推理，建立角色记忆，对玩家不可见。
- **被发现（首回合）**：玩家揭开怪物格时，怪物感知到威胁并进入激活状态，但**本回合不行动**——"感知到"与"做出反应"之间保留一拍延迟。
- **后续每回合**：已激活怪物持续感知玩家的揭格动作，在地下城推进阶段依次行动，行动内容写入游戏日志。

## 界面布局

- **顶部**：状态栏（HP / ATK / DEF）+ 设置按钮。
- **中部**：消息日志（弹性填充剩余空间）。
- **底部**：地图格子区域，锚定底边。
- 设置按钮打开全屏遮罩浮层，点击遮罩或关闭按钮收起。
