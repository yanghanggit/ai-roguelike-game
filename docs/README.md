## 写作约束（AI 编写文档时必须遵守）

- `docs/` 记录的是**相对稳定的设计知识**，不是代码快照。
- 禁止在文档中出现代码块（` ``` `）。类名、函数名等符号用行内代码（`` ` `` ）提及即可。
- 设计意图、架构决策、调用时序等文字描述 > 任何形式的代码示意。
- 每篇文档聚焦单一领域，跨领域引用使用 `[[文件名]]` 或 Markdown 相对链接。
- 文档标题即文件名语义，命名用小写 kebab-case。
